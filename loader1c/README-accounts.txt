===========================================================================
  Loader1C - Маппинг счетов 1С (accounts.config.json)
===========================================================================

ЧТО ЭТО

  accounts.config.json - ручной маппинг банковских счетов на базы 1С.
  Сервис Loader1C использует его, чтобы определить в какую базу 1С класть
  выписку (по номеру счёта получателя/плательщика).

  Пока этот файл существует:
    - сервис НЕ сканирует базы 1С на старте (старт мгновенный)
    - rescan.trigger игнорируется
    - чтобы подхватить изменения - ОБЯЗАТЕЛЬНО перезапустить сервис


ФОРМАТ ФАЙЛА

  JSON с расширениями: разрешены // комментарии и trailing commas.

  {
    "<имя_базы_1С>": [
      { "account": "<номер_счёта>", "org": "<название_орг>", "inn": "<инн>" },
      ...
    ],
    "<следующая_база>": [
      ...
    ]
  }

  Поля:
    account - номер счёта, 18 цифр БЕЗ префикса ME25 и БЕЗ тире
              (минимум 10 символов; не цифровые значения тоже допустимы,
              как например "BIZNESKARTA" - бизнес-карта без обычного №)
    org     - название организации в 1С (для удобства поиска по файлу)
    inn     - ИНН, 8 цифр (для Черногории)

  Имя базы должно совпадать с тем что прописано в:
    C:\scripts\loader1c\appsettings.json   (раздел "Databases")


ПРИМЕР

  {
    "astrasoft": [
      { "account": "560000000000290342", "org": "DOO ASTRASOFT", "inn": "03339645" },
      { "account": "565000000002145048", "org": "DOO ASTRASOFT", "inn": "03339645" }
    ],

    "base_nop": [
      // Норстандарт - две карты + основной счёт
      { "account": "560005010001646782", "org": "NORSTANDART DOO", "inn": "03741621" },
      { "account": "560000000000415763", "org": "NORSTANDART DOO", "inn": "03741621" }
    ]
  }


КАК РЕДАКТИРОВАТЬ

  1. Сделать резервную копию:
       copy accounts.config.json accounts.config.json.bak

  2. Открыть файл в UTF-8 редакторе:
       - VS Code         (рекомендую)
       - Notepad++
       - Sublime Text
       - Modern Notepad  (Windows 10/11 - подходит)
     НЕ используйте старый Windows Notepad на Windows 7/Server 2008.

  3. Добавить / изменить запись.
     - В одну базу можно добавлять сколько угодно счетов.
     - Один счёт = одна организация = одна база.
     - Чтобы убрать счёт - удалите его строку (вместе с запятой).

  4. Сохранить (Ctrl+S). Кодировка - UTF-8.

  5. Перезапустить сервис:
       sc stop Loader1C
       sc start Loader1C

     Или через "Службы" (services.msc) - найти "Loader1C" - перезапустить.

  6. Проверить лог:
       C:\NET\Epsilon\izvod\data\log\loader1c.log

     Должна быть строка:
       "Loaded N accounts from accounts.config.json"

     где N - количество ваших записей. Если меньше чем ожидали - ищите
     в логе предупреждения "Duplicate account" или "Failed to read".


ТИПОВЫЕ ЗАДАЧИ

  --- Добавили нового контрагента (физлицо/компанию) в 1С ---

  Ничего делать не нужно. Контрагент будет автоматически найден или создан
  при загрузке выписки. accounts.config.json содержит только счета НАШИХ
  организаций, не контрагентов.


  --- Завели новый р/с одной из наших организаций ---

  1. Открыть accounts.config.json
  2. Найти секцию с этой организацией (по `org`)
  3. Добавить новую строку с новым счётом, например:
       { "account": "560005000000123456", "org": "DOO ASTRASOFT", "inn": "03339645" },
  4. Перезапустить сервис.


  --- Завели новую организацию в существующей базе 1С ---

  1. Открыть accounts.config.json
  2. Найти секцию с нужной базой (например, "astrasoft")
  3. Добавить запись с новой `org` и её счётами
  4. Перезапустить сервис.


  --- Завели новую базу 1С ---

  1. Открыть C:\scripts\loader1c\appsettings.json
  2. Добавить имя базы в список "Databases"
  3. Открыть accounts.config.json
  4. Добавить новую секцию для этой базы с её счётами
  5. Перезапустить сервис.


  --- Перенесли организацию между базами 1С ---

  1. В accounts.config.json - удалить записи орг. из старой базы
  2. Добавить их в новую базу
  3. Перезапустить сервис.


ОШИБКИ И ОТЛАДКА

  --- Сервис не стартует / стартует но не находит счета ---

  В логе ищите:
    "Failed to read accounts.config.json: <причина>"
      => Синтаксическая ошибка JSON. Проверьте запятые, кавычки, скобки.
         Восстановите файл из .bak.

    "Loaded 0 accounts from accounts.config.json"
      => Файл валиден но пуст или все записи отбракованы.
         Проверьте что account длиной >= 10 символов.

    "Account NNN not found in any database"
      => В выписке есть счёт NNN которого нет в конфиге.
         Добавьте его или проверьте что номер совпадает (без ME25, без тире).


  --- Дублированный счёт в логе ---

    "Duplicate account NNN in config (db=A vs db=B) - keeping first"
      => Один счёт прописан в двух базах. Найдите и удалите дубль.


  --- Не уверены что счёт правильный ---

  В этой же папке есть account_mapping.txt - человекочитаемый отчёт от
  последнего автоматического сканирования 1С. Там можно найти счёт и
  посмотреть в какой базе он живёт.


СПЕЦИАЛЬНЫЕ ОПЕРАЦИИ

  --- Перегенерировать конфиг из 1С ---

  Если хотите начать заново со свежим сканом всех 23 баз:

    sc stop Loader1C
    cd C:\scripts\loader1c
    del C:\NET\Epsilon\izvod\data\accounts.config.json
    Loader1C.exe scan              :: ~9 минут, сканирует все базы
    Loader1C.exe seed-config       :: создаёт новый accounts.config.json
    sc start Loader1C

  ПРЕДУПРЕЖДЕНИЕ: seed-config перезапишет ваш файл - комментарии и порядок
  будут потеряны. Сделайте резервную копию заранее.


  --- Вернуться к авто-скану 1С (отключить ручной конфиг) ---

  Если решили не использовать ручной конфиг, а сканить 1С автоматически
  на каждом старте (когда mapping старше 12 часов):

    del C:\NET\Epsilon\izvod\data\accounts.config.json
    sc stop Loader1C && sc start Loader1C

  Сервис прочитает старый кэш (account_mapping.json) или запустит скан.


  --- Принудительный скан 1С при наличии конфига ---

  Если accounts.config.json есть, но хочется на лету пересканировать 1С
  (НЕ перезаписывая конфиг):

  Создайте триггер - сервис игнорирует его в режиме конфига:
    echo. > C:\NET\Epsilon\izvod\data\rescan.trigger

  В логе будет:
    "rescan.trigger ignored - mapping is config-based"


СВЯЗАННЫЕ ФАЙЛЫ В ЭТОЙ ПАПКЕ

  accounts.config.json     - ручной маппинг (АВТОРИТЕТНЫЙ источник)
  accounts.config.json.bak - резервная копия (создавайте вручную)
  account_mapping.json     - автогенерированный маппинг от Loader1C.exe scan
                             (используется только если accounts.config.json
                             отсутствует)
  account_mapping.txt      - человекочитаемый отчёт от auto-сканирования
                             (создаётся вместе с .json)
  rescan.trigger           - сигнальный файл для запуска авто-скана
                             (игнорируется когда есть accounts.config.json)

  output/                  - входная очередь .txt выписок
  loaded/                  - архив успешно загруженных выписок
  log/loader1c.log         - лог сервиса


КОНТАКТЫ И ССЫЛКИ

  Конфигурация сервиса:    C:\scripts\loader1c\appsettings.json
  Исполняемый файл:        C:\scripts\loader1c\Loader1C.exe
  Логи:                    C:\NET\Epsilon\izvod\data\log\loader1c.log
  Парсер выписок (izvod):  http://localhost:8000
